# Grace 平台：UI 设计与 PRD 需求映射说明 ## 1. 核心模块映射 | PRD 模块 | UI 页面 | 实现逻辑说明 | | :--- | :--- | :--- | | **Video_Manager** | 视频上传、视频管理 | 页面支持断点续传视觉反馈，管理列表展示 MySQL 中的视频基本信息和状态。 | | **Metadata_Generator** | 元数据审核 | UI 展示“AI 生成”标识，支持通过“重新生成”按钮再次触发阿里云 LLM 服务。 | | **Review_Console** | 元数据审核 | 左右分栏布局：左侧预览视频，右侧编辑标题、描述、标签，直接对应需求 3 的验收标准。 | | **Video_Distributor** | 分发配置 | 采用 Step-by-Step Wizard，第一步选视频，第二步选平台（由 Registry 路由），第三步确认。 | | **Channel_Manager** | 设置 (Settings) | “账号绑定”部分处理 OAuth 授权，底层对应 AES-256-GCM 加密存储 API Key。 | | **Promotion_Advisor** | 推广任务 | 展示 AI 生成的定制化文案（POST/COMMENT/SHARE），支持用户确认并触发。 | | **Promotion_Executor** | 推广历史、报告 | 实时展示 OpenCrawl 的执行进度和各渠道结果报告，符合需求 7 的汇总要求。 | ## 2. 关键业务流程对齐 (Acceptance Criteria Check) ### 需求 1：本地视频上传 - **UI 实现**：提供拖拽上传区域，实时显示上传速度、剩余时间和百分比进度条。 - **状态处理**：上传完成后状态流转为“元数据生成中”。 ### 需求 3：用户审核与确认 - **UI 实现**：`Review_Console` 提供了完整的编辑表单。 - **交互逻辑**：点击“确认元数据”后，后台将状态更新为 `待发布`，前端列表同步更新。 ### 需求 5 & 8：可扩展性与渠道管理 - **UI 实现**：`Settings` 页面预留了多平台卡片（YouTube 已激活，抖音/B站/小红书灰度显示），体现了 `Distributor_Registry` 的架构灵活性。 ## 3. 技术栈兼容性建议 - **前端适配**：UI 组件采用响应式设计，方便博主在不同设备上查看推广报告。 - **数据交互**：建议通过 WebSocket 实现视频上传进度和 AI 文案生成的实时推送，以匹配您后端 Java 21 的高性能特性。
